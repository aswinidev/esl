<smart-image> component

Mods

Mode: save-ration

Smart Image use entire available width and occupies the height according to image aspect ratio.

No inner image, background using to render image.

Mode: cover

No inner image, image rendered by background image.

Smart Image have no own size. Can be used with the img-container classes

Example:

<div class="img-container img-container-4-3"> <smart-image ...
Mode: cover (additional classes: vertical alignment)

As the background-image used to render image in cover mode we can control image cropping options throw the css.

The following options available to control image vertical position:

  • .bg-v-top
  • .bg-v-center (default)
  • .bg-v-bottom
Container:size: 400 x 200px; gray border
Mode: cover (additional classes: horizontal alignment)

Same as the following options available to control image horizontal position:

  • .bg-h-left
  • .bg-h-center (default)
  • .bg-h-right
Container:size: 200 x 300px; gray border
Mode: cover (additional classes: inscribe)

bg-inscribe can be used to declare inscribe image behaviour

Container:size: 300 x 150px; gray border
Mode: fit

Smart Image use inner img to render picture and used as a wrapper that controls image source and state.

Mode: origin

Origin mode do the same as fit but set smart-image size according inner image original size

Lazy loading

Mode: save-ration, lazy mode

Lazy marker allows to start image loading only when it is close to the viewport and visible

Mode: fit, lazy mode

Lazy loading available for all rendering mode

SVG Inline loading Beta

Mode: inner-svg, lazy mode

Smart Image inner-svg mode that loading svg content inline.

NOTE: requesting svg endpoint should accept CORS polycy restrictions
